Ocelotds : Ocelot datas services

Welcome to Ocelotds! Ocelotds is java and javascript framework remoting and web messaging technology that enables developers to easily connect to back-end data and push data in real-time to javascript applications, for more responsive rich Internet application (RIA) experiences.

The best and easiest communication way between java 7+ and javascript

Do that in java

@DataService(resolver = Constants.Resolver.CDI)
public class MyService {
   public MyResult getSomething(Param1 p1, Param2) {
      // Your business code
   }
}

Do that in javavascript

myService.getSomething(param1, param2).then(function (myresult) {
   doSomethingWithMyResult(myresult);
});

Forget REST, forget AJAX, forget http, forget protocol, Ocelotds uses a WebSocket and do everything for you.

Ocelotds use one bidirectional connection WebSocket, and is designed to be use in a single page application (SPA).

Forget the browsers connection limits with your backend. The 6 simultaneous connections limit is not your problem anymore.

Ocelotds can share thousands messages between back-end and front-end in seconds.

Great features

Ocelot framework allows javascript calls to different services directly to your java methods call ! As if your were in you back-end.

Don't write WEB Services, focus on business code. Ocelot will manage the communication between the business layer and the front-end.

The best way, is to use managed beans, as EJB, CDI Bean or Spring bean. But you can call a simple pojo.
If you use managed classes, you will benefits from all features

Java EE compliant

Ocelot uses Java EE technologies, nothing else, so you can use all natives and standards features of Java EE.

Example : @Interceptor, @Decorator ...

Comprehensive and very easy

No configuration needed, just add ocelot to your maven dependencies.

Very fast

Ocelot can share thousands of messages in seconds between the java layer and the javascript layer.

Native push messages

Ocelot uses a bi-directional connection. So to write a chat-like, channel or topic paradigm is really easy. Check the Demos, and see by yourself !

Cache features

Configure your cache directly from the back-end : annotate a method with @JsCacheResult and its the result will be cached on the front-end.
To reset the front-end cache, annotate another method with @JsCacheRemove and it will remove the cache for all clients.

Helper page

Ocelotds exposes a page with the totality of services, you can test them easily.

Agnostic

Ocelot can work with the most famous back-end technologies : EJB, CDI, Spring. But also with the most famous front-end frameworks : Angularjs, knockoutjs,...

Extensible

Ocelot is extensible. You can write your own resolver to access to your favorite technologies.

At the moment, Ocelot is tested with Glassfish 4.1, Wildfly 9, and Tomcat 7/8.

Ocelot is developped with Java EE server glassfish 4.
But Ocelot can work in all servlets containers, like tomcat, without EJB features of course. Just add the cdi and json processing implementation :

  • JSR299 - JSR346 : Context dependency Injection for Java : weld
  • JSR353 : Java API for JSON Processing - Reference Implementation : jsonp

See the documentation about the tomcat server for more details.
