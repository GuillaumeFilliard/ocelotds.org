Advanced feature on front-end

Launch services at document loading

Understand that ocelot framework is completly asynchronous.
So if you try to call server before the connection is etablished, there will be probably a problem.
For fix this problem (not really one) the ocelotController have a method for add handlers for when the connection is ready.


ocelotController.addOpenListener(function() {
   QUnit.test("Test MyService", function (assert) {
      var done = assert.async();
      var srv = new MyService();
      srv.getMessage(5).Then(function (msg) {
         assert.equal(msg, "expected");
         done();
      });
   };
}

Optional configuration

minify/magnify javascript

By default javascript ocelot serves ocelot.js minified
This is usefull for preserve the band width.
Ifyou want to read it, magnify it adding minify=false to ocelot.js url


<body>
   <script src="ocelot.js?minify=false" type="text/javascript"></script>

configure stacktrace size sent to client

In your web project, you can specify the size of stacktrace sent on client side.
The default value is 50
If you set 0 the stacktrace will be flush in backend logger.
In deed, you can internationalize the error message easily. Show i18n section i18n.
But for debugging, it's usefull to get the full stacktrace throwed from back-end.
Limit the size in production environnement for preserve the band width.
Add section in the standard web descriptor web.xml


<context-param>
   <param-name>ocelot.stacktrace.length</param-name>
   <param-value>50</param-value>
</context-param>
