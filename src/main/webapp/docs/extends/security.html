Create a new security service

Security mecanism need vendor implementation.

Ocelotds assume implementations for supported servers, if you want to use ocelotds in other server, you have to implements ContainerSecurityServices

Dependency :

<dependency>
   <groupId>org.ocelotds</groupId>
   <artifactId>ocelot-spi</artifactId>
   <version>${ocelot.version}</version>
</dependency>

ContainerSecurityServices :

@ContainerQualifier("MyVendor")
public class MyContainerSecurityServices implements ContainerSecurityServices {
   @Override
   public SecurityContext getSecurityContext() {
   // how to get vendor SecurityContext implementation
   }
   @Override
   public void setSecurityContext(SecurityContext securityContext) {
   // how to set vendor SecurityContext implementation
   }

SecurityContext :

What is SecurityContext

SecurityContext is the empty interface, use it for implement your specific implementation

public interface ContainerSecurityServices {
...SecurityContext getSecurityContext();
...void setSecurityContext(SecurityContext securityContext);
}

Define the vendor container implementation :

By default ocelotds looking for a ContainerSecurityServices qualified by the name of vendor container

The name used for find the vendor name is the name returned by ServletContext.getServletInfo().

By example the specific implementation for glassfish is annotated by :

@ContainerQualifier("GLASSFISH")

ServletContext.getServletInfo() in glassfish return : 

GlassFish Server Open Source Edition  4.1

So server info contains glassfish, and glassfish ilmplementation will be choose.

Define the extra security framework implementation :

If you want use a specific framework, not the container implementation, you have to specify the entry key in web.xml 

<context-param>
   <param-name>ocelot.security.impl</param-name>
   <param-value>MyVendor</param-value>
</context-param>
