Ocelotds : Ocelot datas services

Welcome to Ocelotds! Ocelotds is java and javascript framework remoting and web messaging technology that enables developers to easily connect to back-end data and push data in real-time to javascript applications, for more responsive rich Internet application (RIA) experiences.

The best and easiest communication way between java 7 and javascript

Forget REST, forget AJAX, forget http, forget protocol, Ocelotds uses websocket and do everything for you.

Ocelotds use one bidirection connection websocket, and is designed for usage in single page web application.

Forget browsers limitations about number of connections between browsers and backend. At best 6 simultaneous connections

Ocelotds can share thousands messages between back-end and front-end by secondes.

Open source

 Ocelot is open source MPLv2.
 Fork it or contribute to the project.

Great features

Ocelot framework allow to call differents services directly from simple classe methods call, like you can do in the backend.

Don't write WEB Services, focus on business code, ocelot do communication between business layer and front-end.

The better way, is using managing beans, as EJB, CDI Bean or Spring bean, but you can call a simple pojo.
If you use mananged classes, you benefits of all features

Java EE compliant

Ocelot use Java EE technologies, nothing else, so use all natives and standards features of Java EE.

Example : Roles based annotation, @RoleAlowed...

Comprehensive and very easy

Configuration less, just add ocelot to maven dependencies.

Very fast

Ocelot share thousands messages by secondes between the java layer and the javascript layer.

Native push messages

Ocelot use bi-directional connection, so for write a chat-like or topic paradigm, is really easy. See demos

Cache features

Code cache features directly from the back-end, annotate a method with @JsCacheResult and the result will be cached on the front-end. Annotate a method with @JsCacheRemove and remove cache of an other method for all clients.
