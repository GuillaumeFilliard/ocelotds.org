Why Ocelot.ds?

Forget REST, JSP, Servlet... Call Java services from javascript like your services were writen in javascript.

Forget browers limitations and the 6 connections simultaneous.

Ocelot uses a single websocket connection, and is very efficient.

Ocelot is comprehensive and very easy

Config less, just add ocelot to maven dependecies.

Great features

Very fast

With Ocelot, share several thousands messages by secondes between the java layer and the javascript layer.

Native push messages

Ocelot use bi-directional connection, so for write a chat-like or topic paradigm, it's very easy.

Cache feature

Code cache feature directement from the back-end, annotate a method with @JsCacheResult and the result will be cached on the front-end.

Annotate a method with @JsCacheRemove and remove cache of an other method for all clients.

Agnostic

Ocelot can works with the most famous back-end technologies (EJB, CDI, Spring).

Ocelot can works with the most famous front-end frameworks (Angularjs, knockoutjs...).

Extensible

Ocelot is extensible, write your own resolver for access to your favorite technology.
